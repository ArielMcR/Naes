Tipos

Três pilares da documentação

Cada tipo de requisito tem seu espaço próprio, com campos e categorias adequados.

RF
Requisito Funcional

Define o que o sistema deve fazer — comportamentos, funcionalidades e interações com o usuário.

  • Código automático (RF001, RF002...)
  • Prioridade: Alta / Média / Baixa
  • Status do ciclo de vida
  • Ator envolvido
  • Vínculo com Regras de Negócio
RNF
Requisito Não Funcional

Define como o sistema deve se comportar — qualidade, desempenho, segurança e restrições técnicas.

  • Código automático (RNF001...)
  • Categoria: Desempenho, Segurança...
  • Prioridade e status
  • Usabilidade, Disponibilidade
  • Escalabilidade
RN
Regra de Negócio

Define as políticas e restrições do domínio do negócio que o sistema deve respeitar.

  • Código automático (RN001...)
  • Status e ciclo de vida
  • Vinculável a um ou mais RF
  • Rastreabilidade bidirecional
  • Exportação destacada
Funcionalidades

Tudo que você precisa

Ferramentas pensadas para o dia a dia de analistas e times de desenvolvimento.

Projetos organizados

Crie projetos e estruture os requisitos em arquivos por módulo, sprint ou contexto. Sem limite de projetos.

Vínculo RN ↔ RF

Associe regras de negócio diretamente aos requisitos funcionais. Rastreabilidade completa em um clique.

Ciclo de vida completo

Gerencie o status de cada requisito: Rascunho → Em análise → Aprovado → Implementado → Cancelado.

Priorização

Classifique requisitos por prioridade Alta, Média ou Baixa para orientar o planejamento do time.

Código automático

Cada requisito recebe um código único gerado automaticamente (RF001, RNF001, RN001) ao ser criado.

Exportação profissional

Gere documentos em PDF, planilhas Excel ou texto corrido para compartilhar com clientes e stakeholders.

Como funciona

Do projeto ao documento

Cinco passos simples para documentar requisitos com qualidade.

1
Crie um Projeto

Dê um nome, descrição e defina o status inicial. O projeto é o container principal de tudo.

2
Adicione Arquivos

Organize por módulo ou contexto — ex: "Módulo de Login", "Módulo de Pagamento".

3
Cadastre RF, RNF e RN

Crie requisitos com código gerado automaticamente, defina prioridade, status e detalhes.

4
Vincule Regras aos RF

Garanta rastreabilidade associando cada RN ao requisito funcional correspondente.

5
Exporte a documentação

Gere PDF, Excel ou texto corrido e entregue documentação profissional ao cliente.

Exportação

Três formatos de saída

Exporte por projeto inteiro ou por arquivo individualmente.

PDF

Documento formatado, pronto para apresentar a clientes e stakeholders.

Excel

Planilha estruturada para análise, filtros e controle por parte do time.

Texto Corrido

Formato narrativo para inserção em documentos de especificação ou contratos.
